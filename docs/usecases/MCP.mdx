# Bitquery MCP Server — now live

**Endpoint:** [https://mcp.bitquery.io](https://mcp.bitquery.io)

**Important:** this is **not** an API wrapper. The MCP connects directly to Bitquery’s **ClickHouse** database, which is the same production database that powers Bitquery—so your agent can query outlier-filtered trade data with the tools exposed by the server. 
For graphQL examples of OHLC, aggregates, and related examples, use the [Crypto Price API](https://docs.bitquery.io/docs/trading/crypto-price-api/introduction/) instead; details are in [References](#references).

If you need help, use [our support channel](https://t.me/Bloxy_info) or open a ticket on [support.bitquery.io](https://support.bitquery.io/).

## Setup — Claude Desktop

1. Open **Connectors**, choose **Custom connector**, paste the MCP URL (for example `https://mcp.bitquery.io/mcp`), add a **name**, and click **Add**.

![Add custom connector: Name and Remote MCP server URL fields](/img/mcp/custom_connector.png)

You do not need to pick tools manually in the chat UI—ask in plain language (for example, **"get me bitcoin trade stats"**) and Claude uses Bitquery when your request fits.

## Setup — Claude Code (terminal)

```bash
claude mcp add bitquery -- npx -y mcp-remote https://mcp.bitquery.io/mcp
```

## Setup — Cursor IDE

Add to `.cursor/mcp.json` (create the file if it does not exist):

```json
{
  "mcpServers": {
    "bitquery": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "https://mcp.bitquery.io/mcp"]
    }
  }
}
```

Restart Cursor or reload the MCP configuration so the server is picked up.

## First connection and permissions

After configuration, you should see **Bitquery Trading Data** among your MCP connectors, for example, in Claude Desktop’s connector list:

![Bitquery MCP listed among custom connectors](/img/mcp/mcp-connectors-bitquery-listed.png)

The first time a tool runs, the client may ask you to **connect** and approve the request:

![Tool run requesting Connect for Bitquery](/img/mcp/mcp-tool-connect-prompt.png)

You will be prompted to **sign in** to your Bitquery account (create one free if needed):

![Bitquery sign-in to authorize the application](/img/mcp/mcp-bitquery-sign-in.png)

Then **authorize** the client (for example, Claude) to use your account:

![Authorize Bitquery access for the requesting application](/img/mcp/mcp-bitquery-authorize-access.png)

In clients like Cursor, you can choose whether tool calls need approval each time or are always allowed:

![Tool permission options such as Always allow](/img/mcp/mcp-tool-permissions-dropdown.png)

## Quick test after setup

Ask your assistant something like:

> Show me the top 10 traders on Solana by realized profit in the last 24 hours.

That kind of question can scan large volumes of trades across Solana DEX venues, aggregate by wallet, and rank—directly against ClickHouse through the MCP tools.
